Catalogo de Skills

Cada skill con descripcion, triggers de activacion y ejemplo practico.

🎮
Orchestrator PO
Orquestacion
Meta-skill que coordina flujos de trabajo de PO. Enruta automaticamente a skills de investigacion (Jira, Confluence, Slack, Figma), ejecucion (tickets, sprint, stakeholder), QA (Test Plan Creator) y analytics (Pendo Tracker) segun tu peticion.
flujo complejo investigar + crear coordinar trabajo PO
Ejemplo de uso
"Investiga en Slack y Confluence lo que se decidio sobre la feature X y creame las stories en Jira"
🔎
Jira Researcher
Investigacion
Busca y extrae informacion de Jira con recuperacion progresiva. Navega jerarquias (epics → stories → subtasks), comentarios e issues vinculados sin sobrecargar contexto.
estado del sprint buscar tickets progreso epic JQL blockers
Ejemplo de uso
"Cuantos story points quedan por completar en el sprint actual? Hay algun ticket bloqueado?"
📖
Confluence Researcher
Investigacion
Navega la jerarquia de paginas de Confluence para encontrar specs, docs de arquitectura, notas de reuniones y runbooks. Extrae informacion clave sin volcar paginas enteras.
buscar spec arquitectura meeting notes runbook wiki
Ejemplo de uso
"Que dice la spec sobre el flujo de autenticacion SSO? Necesito los acceptance criteria"
💬
Slack Researcher
Investigacion
Busca y extrae informacion de Slack con recuperacion progresiva. Navega canales, threads y mensajes sin tirar del historial completo de conversaciones.
que se discutio decisiones Slack buscar canal contexto thread
Ejemplo de uso
"Que decidio el equipo sobre el approach de migracion en #eng-platform la semana pasada?"
🎨
Figma Researcher
Investigacion
Extrae contexto de diseno desde Figma via MCP. Recupera screenshots, estructura de componentes, tokens de diseno y specs de layout. Traduce decisiones visuales en specs accionables para ingenieria.
Figma URL check the design UI specs mockup design tokens
Ejemplo de uso
"Que muestra el Figma para el nuevo alert banner? Extraeme las specs y los estados del componente"
🎯
Jira Ticket Creator
Ejecucion PO
Crea issues estructurados en Jira con templates en prosa humana (sin tablas de metadata ni notas de AI). Story segun el template oficial ACME-17271 y Bug segun el esquema oficial ACME-56921. Soporta Epics, Stories, Tasks, Bugs y Vulnerabilities. Tracking de Pendo opcional (opt-in por story) y siempre colapsado. Flujo de 5 pasos: inputs → preview → estimacion → confirmacion → resumen. v2.0.0
crear outcome crear milestone crear epic nueva story reportar bug vulnerability
Ejemplo de uso
"Crea un Milestone para agrupar las epics del nuevo modulo BI y definir sus criterios de exito"
📅
Sprint Planner
Ejecucion PO
Planificacion de sprints, grooming y estimacion. Sprint goals, capacity planning, velocity tracking, priorizacion del backlog y preparacion de ceremonias agile.
planificar sprint grooming velocity capacidad priorizar
Ejemplo de uso
"Ayudame a preparar el sprint planning para Sprint 24. Tenemos 3 devs a 80% capacity"
📣
Stakeholder Communicator
Ejecucion PO
Comunicacion con stakeholders: sprint status updates, release notes, executive summaries, decision logs, escalation reports y preparacion de demos.
status update release notes executive summary preparar demo
Ejemplo de uso
"Prepara el sprint status update para el stakeholder meeting de manana con metricas"
📊
Pendo Tracker
Analytics
Determina que track events de Pendo aplican a una issue de Jira. Genera la seccion de tracking (formato light o full, siempre colapsada) y realiza verificacion doble obligatoria: (A) el PLG Instrumentation Plan en Confluence y (B) los datos reales de Pendo en produccion via searchEntities. Nunca da por confirmado un evento que no exista en Pendo. Funciona standalone o como companion del Jira Ticket Creator.
Pendo track events instrumentacion PLG tracking que trackear
Ejemplo de uso
"Crea una story para el rediseno de la pricing page y asegurate de que trackeemos lo necesario en Pendo"
✅
Test Plan Creator
QA
Crea Test Plans en Jira siguiendo las 21 Guidelines oficiales del equipo de QA de Lansweeper. Formato Gherkin, testing por fases (A–G), tests de seguridad, nivel de automatizacion, criterios de reporte de bugs y consistencia con herramientas hermanas. Template con columnas por rol (Dev 1 / Dev 2 / QE / PO) y flujo de 5 pasos.
crear test plan plan de pruebas QA strategy manual tests
Ejemplo de uso
"Crea un test plan para la feature de exportacion CSV del inventario con cobertura de edge cases"

Tips y Buenas Practicas

Como sacar el maximo partido a tus skills.

🎯 Se especifico con el contexto

En lugar de "crea un ticket", prueba crea una Story en SCAN para implementar filtro de OS con acceptance criteria. Cuanto mas contexto, mejor resultado.

🔗 Encadena naturalmente

Claude detecta cuando necesita multiples skills. "Investiga en Slack que se dijo sobre X y crea tickets" activara Slack Researcher + Jira Ticket Creator automaticamente.

💡 Usa el Orchestrator para flujos complejos

Si necesitas investigar en varias fuentes y luego actuar, usa el Orchestrator PO como punto de entrada. El enrutara automaticamente a las skills correctas.

🔍 Investiga antes de crear

Deja que los Researchers recopilen contexto antes de crear tickets o reports. El resultado sera mucho mas preciso y completo.

✅ No olvides QA

Despues de crear stories con el Ticket Creator, usa Test Plan Creator para generar el plan de pruebas automaticamente basado en los acceptance criteria.

📈 Combina para ceremonias

Para sprint reviews: Jira Researcher (datos) + Sprint Planner (analisis) + Stakeholder Comms (presentacion). Todo en un solo flujo.

📊 Tracking en paralelo con Pendo

Cuando crees stories con cambios de UI, incluye siempre la seccion de Pendo. Usa Pendo Tracker para que te ayude a identificar que eventos del PLG Instrumentation Plan aplican a tu feature.
